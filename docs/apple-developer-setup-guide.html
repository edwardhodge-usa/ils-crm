Create Developer ID Application Certificate
Manual

This certificate lets you sign apps for distribution outside the App Store. It's required for notarization and for your employees to open the app without Gatekeeper warnings.

Open Xcode Settings

Open Xcode (make sure it's the latest version).

Go to the menu: Xcode → Settings (or press Cmd + ,)

Navigate to Accounts tab

Click the Accounts tab at the top of the Settings window.

You should see your ImagineLab Studios team listed. If not, click the + button at the bottom left and sign in with your Apple ID.

Open Manage Certificates

Select your ImagineLab Studios team from the list.

Click the Manage Certificates... button (bottom right of the window).

You should see a list of existing certificates. You likely already have an Apple Development certificate.

Create Developer ID Application certificate

Click the + button at the bottom left of the certificates list.

Select Developer ID Application from the dropdown menu.

Click Create.

This creates a certificate that Apple trusts for apps distributed outside the App Store. It's stored in your macOS Keychain automatically.
If you don't see "Developer ID Application" as an option, your Apple Developer account may not have the right permissions. Make sure you're the Account Holder or Admin for the ImagineLab Studios team.
Verify the certificate was created

Open Terminal and run:

security find-identity -v -p codesigning | grep "Developer ID"

You should see something like:

1) ABCDEF123456... "Developer ID Application: ImagineLab Studios (3CHSW4M2B7)"
If you see the line above, you're good! The certificate is installed and ready to use.

Generate App-Specific Password for Notarization
Manual

Apple's notarization service needs to verify your identity. Instead of using your main Apple ID password, you create a special "app-specific password" just for this purpose.

Go to Apple ID website

Open your browser and go to: https://appleid.apple.com

Sign in with the Apple ID you use for your developer account.

Navigate to App-Specific Passwords

Once signed in, go to: Sign-In and Security → App-Specific Passwords

Click Generate an app-specific password (or the + button).

Create and save the password

When prompted, enter a label like: ILS Notarization

Apple will show you a password that looks like: abcd-efgh-ijkl-mnop

Copy this password NOW and save it somewhere safe (like Notes or 1Password). You will NOT be able to see it again after closing this dialog.
C
Store Notarization Credentials in Keychain
Manual

This saves your Apple ID + app-specific password in the macOS Keychain so the notarization tool can use them without asking every time.

Run the store-credentials command

Open Terminal and run this command (replace the email with your Apple ID email):

xcrun notarytool store-credentials "ILS-Notarize" \ --apple-id "your-apple-id@email.com" \ --team-id "3CHSW4M2B7"

It will prompt you:

  1. App-specific password: Paste the password from Section B (the abcd-efgh-ijkl-mnop one)
When it says "Credentials saved", you're done! The profile name "ILS-Notarize" is what the build scripts will use to find these credentials.
D
Generate Sparkle Ed25519 Keypair for Auto-Updates
Manual

Sparkle (the auto-update framework) uses its own signing system to verify that updates haven't been tampered with. You need to generate a keypair once — the private key stays in your Keychain, the public key goes in the app.

Find the generate_keys tool

First, make sure the project has been built at least once in Xcode (to download Sparkle). Then run in Terminal:

find ~/Library/Developer/Xcode/DerivedData -name "generate_keys" -path "*/Sparkle*" 2>/dev/null | head -3

You should see a path like:

/Users/EdwardHodge_1/Library/Developer/Xcode/DerivedData/ILS_CRM-abc123/SourcePackages/artifacts/sparkle/Sparkle/bin/generate_keys

Copy that full path — you'll need it in the next step.

If nothing shows up, open the ILS CRM project in Xcode, wait for "Resolving Package Dependencies" to finish, build once (Cmd+B), then try the find command again.
Generate the keypair

Run the tool you found above:

/path/to/generate_keys

It will output something like:

A]public key for SUPublicEDKey (Info.plist): dF4b8K9mN2pQ7xY1wR5tH3jL6vC0sA8eG4nB9uI2oP=

The private key is automatically stored in your macOS Keychain (under the service https://sparkle-project.org). You never need to touch it directly.

Copy the public key string (the base64 text). You'll need it in the next step.
Update project.yml with the public key

Tell Claude Code:

"Replace SUPublicEDKey placeholder in project.yml with: [paste your key here]"

Or manually edit swift-app/project.yml and find:

INFOPLIST_KEY_SUPublicEDKey: "PLACEHOLDER_REPLACE_AFTER_KEY_GENERATION"

Replace PLACEHOLDER_REPLACE_AFTER_KEY_GENERATION with your actual public key.

E
Test the App
Manual

Now let's verify everything works together.

Open and run in Xcode

Open swift-app/ILS CRM.xcodeproj in Xcode.

Press Cmd + R to build and run.

The app should:

  1. Show "Verifying license..." briefly
  2. Then go straight to the main UI (because no license PAT is configured yet)
Configure license in Settings

In the running app, go to ILS CRM+ → Settings (or press Cmd + ,).

You should see two new sections at the top:

  1. User — Enter your email address (the one in the Airtable licensing table)
  2. License — Enter the license PAT (ask Claude Code for it, or check the Electron app's license-config.ts)

Click Save License Key.

Restart and verify license check

Quit the app and run it again from Xcode (Cmd + R).

Watch the Xcode console output. You should see:

[app-state] License valid, starting app

The app should load normally after the brief license check.

If you see "License valid" in the console, everything is working! The app now checks your Airtable licensing base on startup and every 2 hours.
!
What Happens Next

Once all the steps above are done, tell Claude Code and it will:

  1. Build a Release archive signed with your Developer ID Application certificate
  2. Package it into a DMG
  3. Submit it to Apple for notarization (takes 1-5 minutes)
  4. Staple the notarization ticket to the DMG
  5. Sign the DMG with your Sparkle Ed25519 key
  6. Create a GitHub Release with the DMG attached

After that, your employees can install the DMG without any Gatekeeper warnings, and the app will auto-update via Sparkle whenever you publish a new release.
